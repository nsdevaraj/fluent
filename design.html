Architecture Overview

This library wraps Fluent UI v9 components with project-specific defaults, validation wrappers, and custom tokens. All styling uses Griffel makeStyles — never plain CSS, Tailwind, or inline styles.

src/
├── components/ui/      ← All wrapped components (one file per component)
├── stories/            ← Storybook stories (one file per component)
├── themes/             ← lightTheme, darkTheme, highContrastTheme
│   ├── lightTheme.ts
│   ├── darkTheme.ts
│   └── highContrastTheme.ts
└── tokens/             ← Custom token groups
    ├── brand.ts        ← Brand color ramp (16 shades)
    ├── borderRadius.ts
    ├── spacing.ts
    ├── typography.ts
    ├── ribbonColors.ts ← 7 ribbon icon color tokens
    ├── dataVizColors.ts← 94 data viz tokens
    └── index.ts

Theming

Wrap your app root with FluentProvider. Three themes available: lightTheme, darkTheme, highContrastTheme.

import { FluentProvider } from "@fluentui/react-components";
import { lightTheme } from "./themes";

<FluentProvider theme={lightTheme}>
  <App />
</FluentProvider>

Which input control to use?

Single value, freeform text? → TextField / Textarea Numeric, bounded range? → SpinButton (increment) or Slider (drag) Select from short list (2–5)? → RadioGroup or Select Select from list, filterable? → Combobox Select from list, no typing? → Dropdown Multiple selections from large set? → TagPicker Date? → DatePicker Time? → TimePicker Search query? → SearchInput

Border Fix Pattern

The Double-Border Problem

Fluent v9 inputs render their bottom accent indicator via a ::after pseudo-element positioned at bottom: -1px (outside the element box). This creates a visible double-dark-line when adding a custom indicator.

Root causes discovered via compiled JS inspection:
1. ::after is positioned OUTSIDE the element — overflow: hidden on the component itself does NOT clip it
2. borderBottomColor on the Input root is set separately from the ::after — overriding one without the other creates visual stacking
3. DatePicker's className routes to the fui-Input span directly — descendant selectors "& .fui-Input" never work

The Fix Pattern

// Wrapper div with clipPath + ::after indicator
<div className={mergeClasses(
  styles.inputWrapper,
  !disabled && validationState !== "error" && styles.inputWrapperIndicator
)}>
  <FluentInput
    className={mergeClasses(
      styles.normalizeInput,   // Reset borderBottomColor to neutral
      styles.killAfter,        // Kill Fluent's built-in ::after
      validationState === "error" && styles.errorBorder,
    )}
  />
</div>
const useStyles = makeStyles({
  inputWrapper: {
    position: "relative",
    clipPath: "inset(0px round 4px)",  // Clips indicator to border-radius
  },
  inputWrapperIndicator: {
    "::after": {
      content: '""', position: "absolute", bottom: "0px",
      left: "0px", right: "0px", height: "1px",
      backgroundColor: tokens.colorNeutralStrokeAccessible,
    },
    ":focus-within::after": {
      height: "2px", backgroundColor: tokens.colorBrandStroke1,
    },
  },
  killAfter: { "::after": { content: "none" } },
  normalizeInput: {
    borderBottomColor: tokens.colorNeutralStroke1,
    ":hover": { borderBottomColor: tokens.colorNeutralStroke1Hover },
  },
});
Why clipPath: inset(0px round 4px) instead of overflow: hidden?
overflow: hidden would clip portal-based dropdowns (Combobox, Dropdown, TagPicker). clipPath does NOT affect portals — they render outside the DOM tree.

Griffel CSS Rules

Griffel uses atomic CSS — never use border shorthand:

// ❌ Wrong
border: `1px solid ${tokens.colorNeutralStroke1}`,

// ✓ Correct — individual properties per side
borderTopWidth: "1px",
borderTopStyle: "solid",
borderTopColor: tokens.colorNeutralStroke1,
// ... repeat for Right, Bottom, Left

Story File Template

import type { Meta, StoryObj } from "@storybook/react";
import { ComponentName } from "../components/ui/ComponentName";

const meta: Meta<typeof ComponentName> = {
  title: "Components/ComponentName",
  component: ComponentName,
  tags: ["autodocs"],
  parameters: {
    docs: { description: { component: "What this component is and when to use it." } },
  },
  argTypes: {
    disabled: { control: "boolean" },
    appearance: { control: "select", options: ["outline", "underline", "filled-darker"] },
  },
};
export default meta;
